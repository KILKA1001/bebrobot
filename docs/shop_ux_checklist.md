# UX-чек-лист магазина (Telegram + Discord)

Проверка перед релизом:

- [ ] **Главный экран:** сразу понятно, что дальше нужно выбрать роль кнопкой.
- [ ] **Карточка роли:** видно цену, описание и следующую кнопку действия «Купить».
- [ ] **Экран подтверждения:** явно указано, что для завершения нужно нажать «Подтвердить покупку».
- [ ] **Ошибка «Недостаточно баллов»:** пользователь видит, сколько нужно и сколько у него сейчас.
- [ ] **Ошибка «Товар недоступен»:** пользователь понимает, что нужно вернуться в магазин и выбрать другой товар.
- [ ] **Ошибка «ЛС закрыты»:** есть понятная инструкция, как открыть личные сообщения и повторить команду.

Примечание по паритету: тексты в Telegram и Discord должны совпадать по смыслу. Различаются только технические нюансы интерфейса (длина текста и подписи кнопок).
