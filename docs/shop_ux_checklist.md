# UX-чек-лист магазина (Telegram + Discord)

Проверка перед релизом:

- [ ] **Главный экран:** сразу понятно, что дальше нужно выбрать роль кнопкой.
- [ ] **Карточка роли:** видно цену, описание и следующую кнопку действия «Купить».
- [ ] **Экран подтверждения:** явно указано, что для завершения нужно нажать «Подтвердить покупку».
- [ ] **Ошибка «Недостаточно баллов»:** пользователь видит, сколько нужно и сколько у него сейчас.
- [ ] **Ошибка «Товар недоступен»:** пользователь понимает, что нужно вернуться в магазин и выбрать другой товар.
- [ ] **Ошибка «ЛС закрыты»:** есть понятная инструкция, как открыть личные сообщения и повторить команду.

## Единая матрица сценариев

| Сценарий | Telegram | Discord | Отличия |
|---|---|---|---|
| Вход в магазин | `/shop` | `/shop` | Нет, единый бизнес-поток. |
| Перенос в ЛС | Из группы бот переносит в личку | Из сервера бот переносит в личку | Только транспортные ограничения платформы. |
| Проверка профиля | До рендера магазина | До рендера магазина | Нет, общая проверка профиля. |
| Выбор категории/товара | Inline-кнопки | `discord.ui.Button` | Только лимиты раскладки кнопок. |
| Пагинация | Callback-страницы | Кнопки назад/вперёд/обновить | Только лимиты кнопок/компонентов. |
| Карточка товара | HTML-текст + кнопки | Embed + кнопки | Только ограничения длины/форматирования текста. |
| Покупка | Подтверждение callback | Подтверждение кнопкой | Нет, единая покупка в shared service. |
| Возврат в магазин | Кнопка «Назад в магазин» | Кнопка «Назад в магазин» | Нет, единая логика возврата в список. |
| Админ-настройки | `/roles_admin` / `/rolesadmin` | `/rolesadmin` | Только формат UI, доменная логика единая. |

Примечание по паритету: тексты в Telegram и Discord должны совпадать по смыслу. Различаются только технические нюансы интерфейса (длина текста и подписи кнопок).
