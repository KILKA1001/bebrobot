# Code-review checklist

Перед слиянием изменений проверяем каждый пункт:

1. **Нет функционального дубля**
   - Перед реализацией проверены существующие модули и хендлеры на наличие аналогичной логики.
   - Если логика уже существует, она расширена или переиспользована, а не продублирована новой реализацией.
2. **Паритет платформ**
   - Для пользовательских сценариев сохранён смысловой паритет Telegram и Discord (различается только UI-слой).
3. **Диагностика ошибок**
   - Для новых веток ошибок добавлено логирование в консоль с контекстом, достаточным для быстрого расследования.
4. **Понятный UX-текст**
   - Пользовательские сообщения объясняют, что делать дальше, и написаны без жаргона.
